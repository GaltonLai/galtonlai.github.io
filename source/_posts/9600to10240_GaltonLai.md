# 【Windows演变史】从Win8.1演变到Win10，曾经存在Win8.2和Win9？

**Author:** GaltonLai

**Date:** 2026-06-24

**Link:** https://zhuanlan.zhihu.com/p/2052905257701742454

大家好，今天想和大家聊聊一个被很多人忽略、却极具历史价值的“技术考古”项目——Windows 8.1到Windows 10之间的演化史。这不是一段简单的版本升级，而是一场微软在操作系统设计哲学上的剧烈震荡与自我重塑。

很多人以为Windows 10是直接从Windows 8“跳”过来的，但事实远比这复杂。在正式命名“Windows 10”之前，微软内部曾经开发过一个代号为“Windows 8.2”“Windows 9”的版本。

Build 9807 - Windows 8.2

![](https://pic2.zhimg.com/v2-ec6053c99abb60d915c3bd098956d627_1440w.jpg)

Build 9834 - 民间上的“Windows 9”

![](https://picx.zhimg.com/v2-a71fe354094fef5ee7c53528d3fa59e9_1440w.jpg)

Build 9860 - 微软引入的“Windows 9”

* * *

**然而，微软还是跳过了“Windows 9”。为什么？**

当时微软跳过Windows 9，有**3种原因**:

1.早期的应用程序检测系统版本时，曾经以“Windows 9”“Windows 9x”字符串检测检测win95/98，所以怕此系统会被误认成老系统而**导致应用崩溃**和**系统不稳定**。

2.市场策略方面，微软吸取了Win8和win8.1反响不佳的教训，**打算系统大改，所以自信跳号**，从Windows 8直接变成Windows 10。

3.其实微软当时还有当初**“多端大一统”**的画饼，也就是**“One Windows”**，所以顺理成章就有了“Windows 10”。

* * *

所以，其实在Build 9841中，微软就引入了名为“Windows 10”的开发计划，但当时是以“Windows Tech Preview（Windows技术预览版）”来命名。

![](https://pic3.zhimg.com/v2-a830086bb1494b7a02e5a4e18dc9ce66_1440w.jpg)

Build 9841

真正有趣的是，这段演化过程并非**一蹴而就**，而是通过数十个内部测试版（Build）逐步完成的。从最早的**Build 9600（即官方RTM版Win8.1）**，到最终的**Build 10240（Win10正式版）**，中间经历了超过20个主要版本迭代。也就是说，每一个Build都像一块拼图，拼凑出我们今天熟悉的Win10界面。

![](https://pic3.zhimg.com/v2-a637787dd969774d87a9d4d325cecca2_1440w.jpg)

Build 9600 - Windows 8.1 RTM

比如，**Build 9780**首次引入了**“窗口化[Metro程序](https://zhida.zhihu.com/search?content_id=277535743&content_type=Article&match_order=1&q=Metro%E7%A8%8B%E5%BA%8F&zhida_source=entity)”**的概念——让Modern应用能以传统窗口形式运行，这是对用户强烈反馈的直接回应。

![](https://pic1.zhimg.com/v2-434d13afdb5287bf99f39774e2f7f452_1440w.jpg)

Build 9780

**Build 9838**则推出了“[吐司通知](https://zhida.zhihu.com/search?content_id=277535743&content_type=Article&match_order=1&q=%E5%90%90%E5%8F%B8%E9%80%9A%E7%9F%A5&zhida_source=entity)”（Toast Notification）和“DockingController”应用，奠定了现代通知中心与多任务管理的基础。

![](https://pic2.zhimg.com/v2-fafc56e1118ee48d8d91709015499bdd_1440w.jpg)

Build 9838

而**Build 9887**开始，系统版本号**正式从NT 6.4升级为NT 10.0**，标志着内核层面的彻底革新。

![](https://pic1.zhimg.com/v2-9ba2addeee642677e513daee1534e9ea_1440w.jpg)

Build 9887

来到了 **Build 9916** - 系统名称**真正引入“Windows 10”**

![](https://pic4.zhimg.com/v2-21a069b2c681dd3cddcbcb17bc0bf52f_1440w.jpg)

Build 9916

**Build 9926**，引入了[UWP框架](https://zhida.zhihu.com/search?content_id=277535743&content_type=Article&match_order=1&q=UWP%E6%A1%86%E6%9E%B6&zhida_source=entity)。

![](https://pic1.zhimg.com/v2-08b19f0d0e6c6b13b891030dce5480cc_1440w.jpg)

![](https://pica.zhimg.com/v2-61aeef052c6a7aa13a2e7b81be55c860_1440w.jpg)

![](https://picx.zhimg.com/v2-e6b3747f4b6086fd0735d71a15df8a71_1440w.jpg)

![](https://pic1.zhimg.com/v2-7a9dcb4095c52ff01e14c273a5c93a14_1440w.jpg)

Build 10154，更新了那个经典的海边巨石壁纸。

![](https://pic1.zhimg.com/v2-1a0678d9a3a37ef04fbb461739ff5ade_1440w.jpg)

**Build 10159**，首次启用了全新的登录界面和桌面背景（Hero背景）——深蓝色光束穿透四格窗格的设计，至今仍是Win10最具辨识度的视觉符号之一。这些看似微小的改动，实则是微软反复权衡“现代感”与“用户习惯”后的结果。

![](https://pica.zhimg.com/v2-5b0f7dc5f73a6046ad2ea942bf9ef7d0_1440w.jpg)

Build 10159 - 登录背景

![](https://pic3.zhimg.com/v2-53fe716f6a0bf4b80dd232576c2a34fc_1440w.jpg)

Build 10159 - 桌面背景

值得一提的是，整个演化过程中，微软大量**借鉴了[Windows Phone](https://zhida.zhihu.com/search?content_id=277535743&content_type=Article&match_order=1&q=Windows+Phone&zhida_source=entity)的设计语言**（如[Cortana](https://zhida.zhihu.com/search?content_id=277535743&content_type=Article&match_order=1&q=Cortana&zhida_source=entity)、动态磁贴），并尝试将**UWP（通用Windows平台）**作为未来核心。但早期版本也充满波折：部分Metro应用无法启动、OOBE（开箱体验）流程不稳定、甚至出现“黑屏启动”等严重Bug。这些“失败”的尝试，恰恰证明了创新的代价与必要性。

最终，在**Build 10176**，一切尘埃落定。它不再叫“Technical Preview”，而是正式命名为“Windows 10”。_系统去掉了时间炸弹，移除了所有测试标记，UI趋于稳定，UWP生态初具规模_。那一刻，微软终于完成了从“触控优先”的激进实验，回归到“兼顾鼠标键盘与触控”的务实路线。

![](https://pic1.zhimg.com/v2-883730d4eafdca232927fd4179fc0d56_1440w.jpg)

Build 10176

最后的**Build 10240**，被定为正式版，也就是我们接触的第一个Windows 10 RTM版本。

![](https://picx.zhimg.com/v2-b11d35a6e1311033841d3af98bde353d_1440w.jpg)

Build 10240 - Windows 10 RTM

**从Build 9600，再到Build 10240。**回望这段历史，可以发现：操作系统不是冰冷的代码堆砌，而是一群工程师在用户抱怨、市场压力与技术理想之间不断妥协与突破的产物。每一个Build编号背后，都是无数深夜的调试、争论与重写。Win10的成功，不在于它有多“新”，而在于它终于学会了倾听——倾听用户的声音，也倾听自己过去犯下的错误。

如果你也曾用过Win8和Win8.1，或许会对这段历程心有戚戚焉；如果你是Win10的长期用户，不妨想想：你桌面右下角那个小小的“Windows”图标，其实承载着一段跌宕起伏的技术史诗。
